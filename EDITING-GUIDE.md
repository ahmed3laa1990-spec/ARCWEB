# ✏️ دليل التعديل اليومي للموقع

> كيف تعدّلون موقع `byarcsa.com` بعد نشره — **بدون الحاجة لأي خبرة برمجية**.
>
> كل تعديل ينشر تلقائياً خلال 30-60 ثانية بعد الحفظ.

---

## 🎯 طريقتان للتعديل

### الطريقة 1: GitHub.com مباشرة (للتعديلات السريعة)
- مناسبة لـ: نصوص، أرقام، روابط، تعديل صغير
- لا تحتاج تنزيل أي برنامج
- تعديل من المتصفح مباشرة

### الطريقة 2: GitHub Desktop + VS Code (للتعديلات الكبيرة)
- مناسبة لـ: إعادة تصميم، إضافة أقسام كاملة
- تنزيل الكود لجهازك، تعديله، رفعه
- نوصي بـ [VS Code](https://code.visualstudio.com) كمحرّر

---

## 📝 تعديل نص بسيط (الطريقة 1)

**مثال:** تغيير "9+ سنوات" إلى "10+ سنوات".

### الخطوات:

1. ادخلوا [github.com](https://github.com) → سجّلوا الدخول
2. افتحوا الـ repo `arc-website`
3. اضغطوا على ملف `index.html`
4. في الزاوية اليمنى العليا اضغطوا أيقونة **القلم (Edit)** ✏️
5. استخدموا اختصار البحث: **Ctrl+F** (أو Cmd+F على Mac)
6. ابحثوا عن: `9+ سنوات` → استبدلوها بـ `10+ سنوات`
7. ابحثوا أيضاً عن النسخة الإنجليزية: `9+ Years` → استبدلوها بـ `10+ Years`
8. اسحبوا لأسفل الصفحة
9. في خانة **Commit changes** اكتبوا وصف التعديل: مثلاً `تحديث سنوات الخبرة لـ 10`
10. اضغطوا **Commit changes** الأخضر
11. ✅ خلال 30 ثانية، التعديل مباشر على `byarcsa.com`

## 🖼️ إضافة صورة مشروع جديد

**مثال:** إضافة مشروع جديد للمعرض.

### الخطوة 1: رفع الصورة

1. ادخلوا الـ repo → افتحوا مجلد `assets/projects/`
2. اضغطوا **Add file → Upload files**
3. اسحبوا الصورة الجديدة (مثلاً `new-project.jpg`)
4. اكتبوا commit message: `إضافة صورة مشروع جديد`
5. اضغطوا **Commit changes**

### الخطوة 2: إضافة بطاقة المشروع في الكود

1. افتحوا `index.html` → اضغطوا القلم للتعديل
2. ابحثوا عن: `<!-- PROJECTS -->` (لإيجاد قسم المشاريع)
3. انسخوا بطاقة موجودة كاملة (من `<div class="project-card"` حتى `</div>` المقابل)
4. الصقوها قبل أو بعد البطاقات الأخرى
5. عدّلوا فيها:
   - `src="assets/projects/1-2.png"` → `src="assets/projects/new-project.jpg"`
   - عنوان المشروع (EN و AR)
   - وصف المشروع
   - الفئة (`data-category="entertainment"` مثلاً)
6. احفظوا Commit

## 👤 إضافة عضو فريق جديد

1. ارفعوا صورة العضو لـ `assets/` (مثلاً `team-ahmed.jpg`)
2. في `index.html` ابحثوا عن `<!-- TEAM -->`
3. انسخوا بطاقة عضو موجودة وعدّلوها:
   - الاسم (EN و AR)
   - المنصب
   - رابط الصورة
   - روابط السوشيال

---

## 💼 إضافة وظيفة شاغرة

1. في `index.html` ابحثوا عن `<!-- CAREERS -->`
2. انسخوا بطاقة وظيفة موجودة وعدّلوها:
   - اسم الوظيفة
   - الموقع (الرياض/جدة/الدمام)
   - نوع التوظيف (Full-time, Contract)
   - رابط البريد للتقديم

## ✍️ نشر مقال مدوّنة جديد

نظام المدوّنة الحالي بسيط (3 بطاقات في الصفحة الرئيسية). لإضافة مقال:

1. في `index.html` ابحثوا عن `<!-- BLOG -->`
2. انسخوا بطاقة مقال موجودة
3. عدّلوا:
   - عنوان المقال (EN و AR)
   - الوصف القصير
   - رابط الصورة (ارفعوها أولاً لـ `assets/`)
   - التاريخ
   - فئة المقال

**ملاحظة:** لمدوّنة كاملة بصفحات منفصلة لكل مقال — أخبروني لأبني نظام مدوّنة احترافي.

## 🎨 تغيير ألوان الموقع

**جميع ألوان الموقع** في مكان واحد. في `index.html` ابحثوا عن:

```css
--navy: #0E2A47;       /* اللون الأساسي */
--gold: #F4B400;       /* اللون الذهبي */
--bg: #FFFFFF;         /* الخلفية */
```

غيّروا قيم الألوان فقط — كل الموقع يتحدّث تلقائياً.

استخدموا [coolors.co](https://coolors.co) لتوليد ألوان متناسقة.

## ↩️ التراجع عن تعديل خاطئ

أعظم ميزة في GitHub — يمكن التراجع لأي إصدار سابق.

### للتراجع عن آخر تعديل فقط:

1. في صفحة الـ repo اضغطوا تبويب **Commits**
2. ستظهر قائمة بكل التعديلات (مع الوقت ومن قام بها)
3. اضغطوا على الـ commit الذي تريدون العودة قبله
4. اضغطوا الزر **... → Revert**
5. ✅ الموقع يعود لحالته السابقة خلال 30 ثانية

### لاسترجاع نسخة قديمة من ملف معيّن:

1. افتحوا الملف (مثلاً `index.html`)
2. اضغطوا **History** في أعلى الملف
3. تصفّحوا الإصدارات → اختاروا الإصدار المرغوب
4. انسخوا محتواه أو اضغطوا **... → Restore this version**

## 👀 معاينة التعديل قبل النشر للعام

Cloudflare Pages يعطي **Preview URL** لكل تعديل. للاستفادة منه:

### طريقة Preview Branches:

1. في GitHub قبل أي commit، اضغطوا **Branch: main** أعلى الملف
2. اكتبوا اسم فرع جديد: مثلاً `test-changes`
3. اعملوا التعديل واحفظوا Commit
4. اذهبوا لـ Cloudflare → Pages → arc-website → Deployments
5. سترون deployment جديد بـ URL مثل: `test-changes.arc-website.pages.dev`
6. افتحوه — هذه معاينة التعديل دون التأثير على الموقع الحقيقي
7. حين ترضون عنه: في GitHub اضغطوا **Compare & pull request** → **Merge pull request**
8. ✅ التعديل ينتقل للموقع الرسمي
